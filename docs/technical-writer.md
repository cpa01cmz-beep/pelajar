# Technical Writer Specialist Guidelines

## Role Overview

The Technical Writer specialist is responsible for creating and maintaining all project documentation to improve Developer Experience (DX).

## Responsibilities

1. **Documentation Creation**: Write clear, concise documentation for all features
2. **Documentation Maintenance**: Keep existing docs up-to-date with code changes
3. **Documentation Standards**: Ensure consistency across all documentation
4. **Knowledge Transfer**: Create guides that help new developers understand the project

## Documentation Types

### User Documentation
- README.md - Project overview and quick start
- API documentation
- User guides

### Developer Documentation
- Architecture docs - System design and patterns
- Contributing guidelines
- Code standards
- Setup guides

### Agent-Specific Documentation
- Agent memory files (e.g., `docs/technical-writer.md`)
- Agent collaboration protocols
- Domain-specific guidelines

## Quality Standards

- Use clear, simple language (suitable for newbies)
- Include code examples where appropriate
- Keep documentation concise but complete
- Use consistent formatting
- Validate markdown syntax

## Workflow

1. **INIT**: Check for existing PRs and issues
2. **PLANNING**: Create work plan for documentation tasks
3. **IMPLEMENT**: Write/update documentation
4. **VERIFY**: Review for clarity and accuracy
5. **SELF-REFLECTION**: Document learnings and create issues if needed
6. **DELIVER**: Create PR with documentation changes

## Best Practices

- Write documentation that you'd want to read
- Assume the reader is a novice developer
- Update docs when code changes
- Review documentation in PRs
- Use visual aids (diagrams, tables) when helpful
