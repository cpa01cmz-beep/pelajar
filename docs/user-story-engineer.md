# User Story Engineer - Domain Guidelines

## Domain Scope
- User story creation and refinement
- Acceptance criteria definition
- Feature specification
- Requirements documentation
- Product requirement alignment

## Issue Creation Standards

### Issue Format
```
Title: [Feature/Enhancement/Bug]: Short description
Label: user-story-engineer
Layer: Product
Owner Agent: user-story-engineer
Problem Statement: What problem does this solve?
Root Cause: Why is this needed?
Proposed Solution: What is the proposed approach?
Acceptance Criteria:
- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3
Definition of Done:
- [ ] User story created
- [ ] Acceptance criteria defined
- [ ] Technical feasibility reviewed
Impact Surface: User | System | DevEx
Priority: High | Medium | Low
```

## Quality Standards
- One issue = one feature/enhancement
- No duplicate scope
- No vague language
- Measurable acceptance criteria required
- Clear user value proposition

## Collaboration
- Coordinate with Product-Architect for alignment
- Work with backend/frontend engineers for feasibility
- Partner with QA for testability review
